# TilePlanner V3.9 技術指南 — 自訂轉角間距

## 1. 核心變革：自訂間距取代自動偵測

在 V3.8 之前的版本，轉角切割的間距是自動抓取系統預設的「灰縫寬度」。然而，在實際施作中，轉角處（尤其是 45 度切角）往往需要與一般面磚不同的間距。

V3.9 正式引入「即時自訂」機制：
- **廢除**：自動抓取預設值。
- **新增**：在選擇收邊形式（45度/蓋磚）後，系統會自動彈出專屬的輸入視窗。

## 2. 動態 WPF 介面實作 (No-XAML approach)

為了保持外掛部署的最高通用性，V3.9 的輸入視窗 `CornerGapDialog` 採用了 **純 C# 動態生成** 技術：

### 技術特點
1. **零檔案依賴**：不需額外的 `.xaml` 檔案，所有佈局（Grid, StackPanel）與控制項（TextBox, Button）均在程式碼中實例化。
2. **UX 優化**：
   - **自動全選**：視窗開啟時，預設值 `1` 會自動被全選，使用者可直接鍵入新數值覆盖。
   - **鍵盤連動**：支援 `Enter` 鍵直接提交（IsDefault = true），實現快速切換。
   - **防呆校驗**：內建 `double.TryParse` 與正數檢查，確保傳入 Revit 核心的參數正確無誤。

## 3. 幾何運算邏輯

使用者輸入的 `GapValue` (mm) 會即時轉換為 Revit 內部的 `Feet` 單位：

- **45度切角 (Miter)**：
  - 兩側磁磚各向內退縮 `gap / 2`。
  - 切割面過 O 點並沿著角平分線方向偏移。
- **蓋磚 (Cover)**：
  - A 側（蓋人側）保持完整（僅極微量容差切割以清理邊緣）。
  - B 側（被蓋側）從交接內角 I 點向後退縮完整 `gap` 距離。

## 4. 部署資訊 (V3.9)

- **主程式檔案**：`ManualCornerJoinCommand.cs` 已更新。
- **無需額外配置**：由於介面動態生成，您不需在 `TilePlanner_V39` 資料夾中放入任何 XAML。
